Git撤销一次合并操作

3月 23, 2017

平时用Git命令行用得比较多， 还以为够熟悉了， 结果今天发生的事， 却让我再一次意识到， 我还差得远呢。

事情的起因是这样的， 我们的官网有这样几个主分支， master, beta, dev, dev是开发分支， beta是对应的内测版本，master对应的正式版本。平时开发的时候都是从dev新开功能分支或者bug分支， 完成之后提PR到dev, 再提PR到beta, 当beta发生pull request事件的时候， 将会触发GitHub的webhook, 然后jenkins进行自动构建与部署。

这一次的问题是， 小伙伴把新的功能分支合并到beta之后， 测试出了一些问题， 需要撤回上一个版本。那么在只能够使用PR来触发更新的情况下， 应该怎么做呢？

首先我自然是想到了屡试不爽的git reset --hard <commit>了， 然而这一次并不能成功， 因为使用了reset之后， 并没有产生新的commit, 然后在提PR的时候， 就提不了。 当然，我也想过直接操作beta分支， 在beta上进行reset, 但是这样一来就不能触发自动构建了。

后来搜了一下， 找到了git revert, 但是这个命令自己压根儿就没使用过， 用起来也是很懵逼的， 我想要像使用git reset那样来用它， 结果报了错， 告诉我缺少-m参数。

呐， 我参考的文章是这个

文章里面讲解的已经比较详细了， 那我这里就不多作介绍了。

（这篇日志真的好水啊😴😴😴）

总之， Git这东西， 还是得好好学才行o(╯□╰)o 。
